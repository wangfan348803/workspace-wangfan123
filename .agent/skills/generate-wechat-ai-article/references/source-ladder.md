# Source Ladder

## Purpose

这份参考文档用于约束联网取源顺序、事实可信度判断和表达边界。

## Priority Levels

### P0 Official

优先级最高。包括：

- 官网与官方新闻稿
- 官方博客
- 官方文档
- 官方 GitHub 仓库、release、项目页
- 官方社媒账号
- 监管机构、标准组织、交易所、法院等原始发布方

优先用途：

- 确认是否真的发生
- 确认发布时间、版本、功能、参数、主体名称
- 确认官方已经公开承认的信息

### P1 High-Quality Secondary Sources

包括高质量行业媒体、主流科技媒体、专业数据库、研究机构、知名分析机构。

优先用途：

- 补充官方没有展开的背景
- 整理时间线
- 交叉验证行业影响和市场反应

### P2 Supplemental Search Results

包括搜索结果中排名靠前但可信度不如前两级的材料。

优先用途：

- 补足边缘细节
- 定位原始信号
- 对已有说法做补充性核对

`P2` 不应用来承载文章最关键的事实判断。

## Retrieval Order

严格按下列顺序：

1. 先查 `P0`
2. 只在 `P0` 不足时查 `P1`
3. 仍有缺口时才查 `P2`

如果 `P0` 已足够支撑核心事实，就不必为了“看起来全面”继续堆 `P1/P2`。

## Fact Confidence Rules

### Can Be Written As Fact

满足任一条件即可：

- 有明确 `P0` 支撑
- 至少两个独立 `P1` 对同一事实给出一致信息，且没有更高等级来源冲突

### Must Be Written Cautiously

出现以下任一情况，必须降级表达：

- 只有单一 `P1`
- 只有 `P2`
- 不同来源表述不一致
- 关键数字、时间或主体无法被原始来源确认
- 属于动机、前景、市场效果、内部决策等推断性内容

可使用的审慎表达：

- “目前公开信息显示”
- “从现有资料来看”
- “媒体报道提到”
- “外界普遍关注的是”
- “这可能意味着”
- “仍需等待更多官方信息确认”

## Conflict Handling

若来源冲突：

1. 优先记录 `P0`
2. 说明冲突点是什么
3. 不强行替读者下结论
4. 在正文或结尾说明截至当前公开资料仍存在不确定性

## Minimum Fact Sheet

写作前至少确认：

- 事件名称
- 主体名称
- 发生日期或时间范围
- 已确认事实
- 未确认但常被提及的说法
- 每个关键点对应的最高等级来源

推荐内部记法：

| Claim | Highest Source | Status | Notes |
|------|----------------|--------|-------|
| 某模型已发布 | P0 | confirmed | 官方博客 |
| 融资金额为 X | P1 | likely | 两家媒体一致，暂无官方披露 |
| 将改变行业格局 | analysis | interpretive | 不是事实 |

## Writing Boundary

事实层不要写成评论。

分析层不要冒充事实。

如果公开资料明显不足，宁可写得保守，也不要把文章凑成一个看似完整但证据不足的结论。
